The pipeline

Four workflows, one bot.

Each stage below is a real n8n workflow built by one team member. This is the order data actually flows through — from a messy CSV to a reply in your chat.

01
Cheang Chee Xuan · Ingestion & Sync

Getting your schedule in, cleanly

Raw school schedules or appointment details are structured before the AI ever touches them — duplicates get filtered out so the calendar stays trustworthy.

Manual Data Entry / CSV Upload → Filter Out Duplicate Fields → Create/Update Google Calendar Event
Slot: optional — paste a screenshot/link of your n8n ingestion workflow here.
02
Chua Jiaqi · AI Core Orchestrator

Reading the calendar, thinking it through

The main processing engine. The AI Agent node is given system prompt instructions and a tool to look up calendar parameters automatically, then figures out what's actually urgent.

Google Calendar Trigger → Advanced AI Agent Node (Prompt Design) → Google Calendar API Read Tool Node
Slot (required): add a "Respond to Webhook" node returning {"reply": "..."}, then paste the webhook URL into N8N_WEBHOOK_URL in script.js.
03
Avya Kesharwani · Front-End & Response Formatting

Turning raw AI text into something readable

This site is that layer — instead of formatting the AI's reply for Telegram, it's formatted and rendered as chat bubbles right here on the web, so there's no separate app to open.

Receive AI Content Payload → Format for the web (this site) → Render in Chat page
Slot: this is your part — chat.html + the chat styling/logic in style.css / script.js. Tweak copy and bubble styling freely.
04
Carissa Chng · QA & Error-Handling Safety Net

Catching it when something breaks

If a node fails or bad data comes through, the workflow catches it and logs a clear error — instead of the bot just going silent.

n8n Error Trigger Node → If/Else Condition Check → Alert Node (Admin Channel)
Slot: the matching front-end fallback lives in fallbackReply() inside script.js — edit that copy to match your QA messaging.
